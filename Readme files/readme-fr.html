INTRODUCTION
PHP-Fusion est un système de gestion de contenu (CMS) ultra-léger et open-source, écrit en PHP 5. Il utilise une base de données MySQL pour sotcker le contenu de votre site et inclut un système d'administration simple et complet. PHP-Fusion comprend également les options que vous trouveriez dans la plupart des autres CMS.

INSTALLATION
Avant d'installer PHP-Fusion vous devez vérifier les conditions requises pour cette version:

- PHP 5.1.2 ou plus
- MySQL 4.1 ou plus

vous devez créer une base de données MySQL. Vous pouvez faire ceci depuis le panneau de contrôle de votre hébergeur web ou via phpMyAdmin. Vérifiez que vous avez à portée de main les informations nécéssaires, en particulier le nom de l'hôte, le nom d'utilisateur, le mot de passe et le nom de la base pour les fournir durant la procédure d'installation.

1. Avant d'uploader les fichiers, renommez le fichier _config.php (situé dans le répertoire files) en config.php.

2. Uploadez le contenu du répertoire files sur votre hébergement web.

3. Sauf si vous installez PHP-Fusion sur un serveur local, vous devez changer les droits (CHMOD) des fichiers et répertoires suivants en 777 :

  • administration/db_backups/
  • downloads/
  • downloads/images/
  • downloads/submissions/
  • downloads/submissions/images/
  • forum/attachments/
  • ftp_upload/
  • images/
  • images/imagelist.js
  • images/articles/
  • images/avatars/
  • images/news/
  • images/news/thumbs/
  • images/news_cats/
  • images/photoalbum/
  • images/photoalbum/submissions/
  • config.php

Note: Certains hébergeurs interdisent le CHMOD 777, veuillez dans ce cas utiliser CHMOD 755.

4. Allez sur votre site, où le fichier setup.php devrait démarrer automatiquement. Si ce n'est pas le cas, exécutez setup.php manuellement en saisissant l'adresse complète de votre site, suivie de /setup.php. Par exemple : http://www.votresite.fr/setup.php.

5. Complétez le processus d'installation en suivant toutes les étapes et en répondant aux questions.

6. Immédiatement après la fin de l'installation de PHP-Fusion, vous devez changer les droits (CHMOD) du fichier config.php en 644 ET effacer le fichier setup.php de votre serveur. Ignorer cette étape pourrait mener à une prise de contrôle de votre site par une autre personne.

MISE A JOUR DEPUIS UNE VERSION V7.00.XX OU V7.01.XX
Avant toute mise à jour, nous vous conseillons fortement de sauvegarder vos fichiers et votre base de données car PHP-Fusion 7.02 est une mise à jour majeure par rapport à PHP-Fusion 7.01. Vous devez suivre précisément les instructions suivantes. Nous vous recommandons de mettre à jour de la version 7.00.xx à 7.01.xx avant de mettre à jour en 7.02.00.

Avant la mise à jour, vous devez vérifier que vous disposez de ces versions:

- PHP 5.1.2 ou plus
- MySQL 4.1 ou plus

1. Le code de la version 7.02 est largement compatible avec celui de la version 7.01, néanmoins il y a eu de nombreux changements dans le noyau et certaines extensions (mods, infusions, paneaux ou thèmes) pourraient ne pas fonctionner correctement.

2. Vous devez tout d'abord uploader le script de mise à jour depuis le répertoire nommé 'upgrade v7' (ou 'upgrade v701' si votre site est déjà en 7.01) dans le répertoire administration de votre site et uploader le répertoire nommé 'locale' dans la racine de votre site. Sans le répertoire 'locale', certains noms pourraient ne pas être ajoutés à la base de données.

3. Connectez-vous à votre site en tant que Super Administrateur. Depuis l'onglet Configuration Système dans le panneau d'Administration, cliquez sur Mise à jour, puis cliquez sur le bouton Mise à jour. VOUS DEVEZ absolument terminer la mise à jour en priorité ! Le processus de mise à jour sera terminé uniquement lorsque vous verrez le message 'Database upgrade complete'.

4. TRES IMPORTANT : Puisque que cette version contient de nombreux changements structurels, certains éléments de votre site ne fonctionneront pas correctement tant que vous n'aurez pas mis à jour tous vos fichiers. VOUS DEVEZ uploader TOUS les fichiers contenus dans le répertoire 'files', exceptés :

  • _config.php
  • setup.php
Changez les droits (CHMOD) des fichiers et répertoires suivants en 777 :
  • downloads/
  • downloads/images/
  • downloads/submissions/
  • downloads/submissions/images/
  • ftp_upload/

Si malgré tout vous avez des doutes, n'hésitez pas à vous rendre sur un de nos sites de support pour obtenir de l'aide, il y a de nombreux utilisateurs dans notre communauté qui pourront vous faire part de leur connaissance pour vous aider ou vous conseiller concernant le processus de mise à jour.

Correction mineure pour les thèmes non officiels
Si vous utilisez un thème non officiel pour votre site, vous devrez modifier le fichier theme.php et styles.css afin que les options apparaissent dans le thème.

Images de catégories de nouvelles:
  • Ouvrir themes/votre_theme/theme.php
  • Dans la fonction render_news
  • Chercher ".$news."
  • Remplacer par ".$info['cat_image'].$news."
Commentaires d'avatar:
  • Ouvrir themes/votre_theme/theme.php
  • Dans la fonction render_comments
  • Ajouter if ($settings['comments_avatar'] == "1") { echo "".$data['user_avatar']."\n"; }
  • N'oubliez pas d'ajouter dans le fichier styles.css la classe .comments_avatar { } et personnalisez-la selon vos gouts.
Enregistrez le fichier et uploadez le.
CONSEILS DE SECURITE
Voici quelques conseils utiles pour sécuriser votre site :
  • Vérifier que le fichier config.php n'a pas d'autorisation en écriture (CHMOD 644).
  • Ne jamais laisser le fichier setup.php sur le serveur après avoir terminé l'installation de PHP-Fusion.
  • Mettre des mots de passe différents pour le FTP et la base MySQL.
  • Ne jamais autoriser des pièces jointes exécutables dans les forums, comme des fichiers php, html, exe, ou texte.
  • Utiliser uniquement des extensions qui ont été testées et approuvées comme sécurisées.
